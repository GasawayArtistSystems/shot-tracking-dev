Database Tables

This will list all the tables in the database for Shot Tracker.

asset_step_assignments

Where each step of the asset approval process is. User assigned, status, due date.

asset_steps

When an asset is assigned to a film, it's setup here. Start date, due date.

assets

When an asset is created, it's stored here. Name, category. This triggers asset_steps to be filled out.

assignment_config_presets

Dealing with the assignment config. This stores any configs the assignment has. Assignment_name, rigs, camera, filename. This stores both assignment both the draft and the assignment_config files.

assignment_progress_steps

Stores the progress steps for each assignment. Assignment_id and step_id.

assignments

Stores the name of the assignment and what class it's a part of. Start date, completion date, parent_step_id, and progress_step_id.

bug_reports

Where all the text for the bug reports are stored. Title, description, email, department, area, priority.

class_enrollments

Connects the student with the class and semester. Class_id, user_id, semester_id

classes

Stores all the classes. Name, Code, class number, archived, semester_id, instructor_id

film_crew

Lists all the users in each film crew and what group they are in. Film_id, user_id, group_id

film_step_progress

Used only for pre-production. Shows what step the pre-production is in. Film_id, step_id, node_id.

films

Lists the names of the films. Name, description, semester_id, director_id, upm_id, step_id

groups

All the groups created for the app and what permission level that group has. Name, permission_level, section.

individual_assignment_statuses

Stores all the assignment statuses for every student, every step. Individual_assignment_id, step_id, current_status.

individual_assignments

Every individual assignment for every student. This is the main table that a lot of other tables play off. Assignment_id, users_id, name, start_date, completion_date, archived.

links

Where every link for the workflows and nodes are stored. This shows any crossflows as well. Parent_node_id, child_node_id, step_id, to_flow_id.

nodes

Every single node in every workflow. This shows where they are in the hierarchy and in the y position in the workflows. Colors and completion percentage as well. Name, step_id, position, completion_percentage, color.

preproduction_steps

Each step for each film preproduction steps. This is how the dropdowns are built. Film_id, step_name, start_date, end_date, color.

production_steps

Each step for each film production steps. This is how the dropdowns are built. Film_id, step_name, start_date, end_date, color.

scene_progress_steps

Only the progress step of each scene in thumbnails. Scene_id, step_id, status, assigned_to, due_date.
