🧠 Admin

This page will show the inner workings of the application. This is where the Users are added and edited, Workflows are added, deleted, locked, and edited, and Archives are backed up.

Users

All users of the application must be added and edited here.

Each user has a unique name, username (not shown), email, and groups designation.

Adding Users

To add a new user, click on the + Add New User button. This will bring up an Add New User window.

This window let's the user put in the following values:

  • Name - Users full name
  • Username - usually the 6+2 used in all UC logins
  • Email - use the same username for simplcity
  • Constructed Email - This is created by the app based on the fields entered by the user.
  • Password - use strong password
  • Confirm Password - use same strong password as above
Assign Roles

Groups. The user can choose different groups to be a part of.

The groups are divided in two groups: Classes and Films. The user can choose whichever group they want to be in.

💡BONUS TIP: Students MUST choose Artist in order to be chosen for films.
Editing Users

The Edit User window is exactly like the Add User window.

Users can change any aspect including the Password.

Deleting Users

To delete a user, click on the Delete button next to the users name.

💡WARNING: This is undoable!

Workflows

The heart of the app is the workflows. A user can setup the step order of an entire workflow and what the statuses may be at each step.

Definitions

Overall Flow - The top level of workflows. It has multiple Individual Flows to make the Overall Flow.

Individual Flow - A collection of nodes (steps) that create an entire flow.

Nodes - a step in the process of an individual flow.

Overall Flow -- Individual Flow -- Nodes

An Example

  • Overall flow - Basic Assignment
  • Individual flows - Assignment -- FB Assignment -- Grade-Assignment
  • Nodes (for Assignment) - Standby -- Ready to Start -- In Progress -- Submitted -- Graded

Rules

In order to make the rest of the application work correctly, there are a few rules that must be followed, when creating workflows. All the workflows that are already created follow these rules.

  1. Use of FB - when creating a feedback individual flow, start with FB.
  2. Y position of nodes - the nodes are ordered by both being linked and by y position. This is how the nodes are listed in the dropdowns.
  3. Use Crossflows - when one node affects the node of another individual flow, the user must use crossflows.
  4. Save Flowchart - when adding, editing, or moving nodes, be sure to Save Flowchart

Individual Flows

This is the top level of the workflows. This is what is used for assignments, shots, and assets.

Flow Names- The names of each Flow

Lock - The flow is locked and must be unlocked to edit or delete

Edit/Delete - The edit button brings up an edit modal. The delete will delete the overall flow.
